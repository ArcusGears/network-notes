Lecture
  • Multipul ways to itentify something. In certian situations one way might be more better than another to use to identify.
  • hostname:Mnemonic way to identify an address. Useful for people, but not computers. Tells little about computer/server's location. Made of variable alphanumeric characters.
  • IP addres:IP numbers are the social security numbers to websites. Decimal formatted, 4 numbers, 0-255 seperate by a period. (ie. 0.255.16.255). Heirarchical, read left to right.
  • DNS:(Domain Name System)A data base that links hostnames to IP addresses. Distributed database and an API. Berkeley Internet Name Domain (BIND) uses the DNS. Runs on UDP transport protocol on port number 53 (love bug: 53). Adds delay, usually short, sometimes a little longer. Usually delay is shortened by nearby lower level DNS servers cacheing commonly used urls for fast lookup. Used indirectly.
  • cache: Local memory that stores what you looked up last. Used everywhere to speed up preformance.
  • Host aliasing: A DNS service. When there are multipul host names that point to an IP address it makes sure that they both direct to the correct location or the canonical hostname.
  • Mail server aliasing: A DNS service. Allows the lookup of the canonical hostname of the email address.
  • Load distribution:A DNS service. For large websites with multipul IP addresses it roates which address is used to avoid overloading one server. This balances the amount of users requesting each server.
  • DNS Problems, for simple centralized design:
    • Single point failure: If the DNS server crashes so does the internet (assuming one server).
    • Traffic volume: One server would have to handle all traffic, would become incredibly slow.
    • Distant centralized database: Delays created by DNS server being physically far away.
    • Maitenance: Would have to contain addresses and hostnames for the whole internet, would be huge and require frequent updates.
  • Alternate
    • Root DNS servers: Main root servers that contain whole or most of DNS. Multipul with lower level servers below them. There are 13 root servers atm.
    • Top-level domain (TLD) servers: Responsible for top-level domains, which as .com, .org., .net, .edu, and .gov and country level domains.
    • Authoritative DNS servers: Organizations on the internet had an authoritative DNS server that houses the records of their host names and IP addresses. Such as universities and large companies.
  • DNS message
    • Identification (2 bytes)
    • Flags (2 bytes)
    • # of Questions (2 bytes)
    • # of answer RRs (2 bytes)
    • # of authority RRs (2 bytes)
    • # of additional RRs (2bytes)
    • Questions
    • Answers
    • Authority
    • additional info
  • DNS vulnerabilities
    • DDoS attacks: either sending packets to the root servers in large amounts to slow them down and stop replies. Or send many DNS queires to top-level-domain servers to slow them down.
    • Man in the middle: Intercepts queries from hosts and returns bogus replies. This redirects people to false websites.
    • DNS poisioning: Sends a bogus reply to the DNS server so it caches the false records.
    • Use DNS to create a DDoS against a server, by sending queires to many authoritative DNS servers, each having the spoofed source address of the target host. If packets are larger in bytes than query then they can potentially overwhelm a target.
  • Wireshark
    • Packet sniffing softwear
    • Displays info such as source address, destination address, frames, contents, protocol, etc.
    • Allows you to see traffic between computer and other hosts on the internet
    • Allows you to filter packets by protocol, ip addresses, etc. In real time.

Definitions

  • Communication link: connection that allows transfer of information between hosts and end systems.
  • Packet switches: groups all transmitted data into suitably sized blocks, called packets, which are then transmitted.
  • Transmission rate: speed in bits/second that data is passed over a link.
  • Packet: package of information sent over a link. May be the whole set, or a part of the total data.
  • Link-layer switch:
  • Router switch:
  • Route/Path: sequence of communication links and packet switches that a packet goes from end system to end system.
  • Internet Service Provider (ISP): a network of packet switches and communication links. Provide connection to end systems and content providers.
  • TCP/IP: Specifies the format of the packets that are sent and recieved among routers and end systems.
  • Distributed applications: Have multiple end systems that exhchange data with eachother.
  • Application Programming Interface (API): specifies how a program running on one end system ask the Internet infrastructure to deliver data to a specific destination program running on another end system.
  • Protocol: Defines the format and the order of messages echanged between two or more communicating entities. Also the actions taken on the transmission or receipt of a message/event.
  • Client: Host/end system that is usually a desktop, mobile PC or other smaller computer system.
  • Server: Host/end system that is usually a more powerful machine that stores and distributes webpages, streams videos, emails, ect.
  • Physical medium: Allows a bit to be sent by propagating electro magnetic waves or optical pulses. Examples include twisted-pair copper wire, coaxial cable, multimode fiber-optic cable, terrestrial radio spectrom, and satellite radio spectrum.
  • Guided media: Type of physical medium, waves are guided along a solid medium.
  • Unguided media: Type of physical medium, waves propagate in the air or outer space.
  • Geostationary satellites: Permanently remain above the same spot on Earth. High orbit, which means longer delay.
  • Low-earth orbiting satellites: Much closer to Earth and are not stationary. Less delay, but requires more of them. Can communicate between eachother.
  • Store-and-foreward transmission: Type of packet switch, must recieve the whole package, before it can start to sent the first bits of hte packet onto the outbound link.
  • Output buffer/output queue: Stores packets that the router is about to send to a link. In a store-and-foreward system.
  • Packet loss: If the output buffer is full an incoming packet may be dropped.
  • Forwarding table: Maps destinations addresses to the outbound links of the router
  • Circuit switching: Resources needed along a path for communication between end systems are reserved for the duration of hte communication. Wasteful due to silent periods when dedicated circuts are silent.
  • Packet Switching: Resources are not reserved, and a session's messages use the resources on demand. As such messages/packets may have to wait for access to a link or other resources.
  • Frequency-division multiplexing: (FDM) Frequency spectrum of a link is divided up among the connections established across the link.
  • Time-division multiplexing: (TDM) Time is divided into frames of fixed duration and each frame is divided into a fixed number of time slots. Network dedicates one time slot in every frame to the connection. Dedicated to the sole use of that connection.
  • Customer: Access ISP.
  • Provider: Global transit ISP. Network of routers and communication links that spans the globe, and has at least one router near the access ISP.
  • Regional ISP: An ISP in a region that all access ISPs in that region connect.
  • Teir-1 ISP: large ISP network, similar to global ISP concept, but not as large. Regional ISPs and come access ISPs connect to teir-1 ISPs.
  • Points of Presence (PoPs): Group of one or more routers, in the same location, that allow customer ISPs in a provider's network to connect to ther provider ISPs.
  • Multi-home: When an ISP connects to two or more provider ISPs.
  • Peer (connection): ISPs at the same level of heirarchy connect their networks together directly, so that the traffic between them passes via direct communicaion rather than over upstream intermediaries
  • Internet Exchange Point (ISP): Meeting point where multiple ISPs can peer together.
  • Nodal processing delay: Time required to examine the packet's header to determine where to direct the packet. Also time to check for bit-level errors.
  • Queuing delay: The delay that a packet experience when it waits to be transmitted onto the link/node. Depends on the number of earlier arrived packets.
  • Transimission delay: Packets can only be transmitted after all the packets that arrived before it have been transmitted. Transimission delay is the length of the packet in bits (L) and the transmission rate from router to router in bits/second (R), this it is expressed as L/R. Or the amout of time required to transmit all of the packet's bits into the link.
  • Propagation delay: Time required to propagate from the beginning of the link to the end router.
  • Traffic intensity: With a = averate rate of packet arrival in queue, R = transmission rate, L = length of bits of packets (all), then traffic intensity is equal to (La)/R. If traffic intensity > 1 averate rate of bit arrival exceeds rate of transmission. If traffic intensity <= 1 there is no queing delay as all packets are transmitted before arrival of the next.
