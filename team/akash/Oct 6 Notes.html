File Transfer: FTP

  • HTTP and FTP are file transfer protocols with similar characteristics (both run on top of TCP)
  • Differences:
    • FTP uses two parallel TCP connections (a control connection and a data connection) to transfer a file.
    • Since FTP uses a different control connection it is said to send control information out-of-band. HTTP meanwhile send this information in-band.
      • A control connection sends information such as user identification, password, commands to change remote directory, and commands to put/get files.
      • Data connection is used to actually transfer the file
  • Commands sent in 7-bit ASCII format (people readable)
    • USER: send user information to the server
    • PASS: Send user password to server
    • LIST: retrieve lists of file in the directory
    • RETR: retrieve file from directory
    • STOR: store file in directory

Electronic Mail in the Internet

  • User agents: allow users to read, reply to, forward, save and compose messages. Like Microsoft Outlook or Apple Mail.
  • Mail Server: Forms the core of the email infrastructure. This where mailboxes are stored which are responsible sending and receiving emails.
  • Simple Mail Transfer Protocol (SMTP): Principal application layer protocol for email. It uses the reliable data transfer service of TCP to transfer mail. It has two sides: a client side and a server side.
o Restricts all mail messages to 7-bit ASCII.
